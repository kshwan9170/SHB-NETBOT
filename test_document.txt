# 네트워크 문제 해결 가이드

## 일반적인 네트워크 문제 진단 절차

1. 물리적 연결 확인
   - 모든 네트워크 케이블이 제대로 연결되어 있는지 확인
   - 스위치, 라우터 등 네트워크 장비의 전원이 켜져 있는지 확인
   - 포트 표시등이 제대로 점등되어 있는지 확인

2. IP 설정 확인
   - 정적 IP 설정이 올바른지 확인
   - DHCP가 제대로 작동하는지 확인
   - 기본 게이트웨이 설정 확인

3. 명령줄 도구를 사용한 진단
   - ping: 기본 연결 테스트
   - traceroute: 패킷 경로 추적
   - nslookup: DNS 확인
   - ipconfig/ifconfig: 네트워크 인터페이스 정보 확인

4. 방화벽 및 보안 설정 확인
   - 방화벽 규칙이 네트워크 트래픽을 차단하고 있지 않은지 확인
   - ACL(Access Control List) 설정 검토

## 자주 발생하는 문제 및 해결 방법

### 인터넷 연결 안됨
1. 모뎀/라우터 재시작
2. ISP 서비스 상태 확인
3. DNS 서버를 8.8.8.8(Google) 또는 1.1.1.1(Cloudflare)로 변경

### 네트워크 속도 저하
1. 대역폭 사용량 확인
2. 네트워크 케이블 상태 점검
3. 컴퓨터/서버의 리소스 사용량 확인

### 특정 서비스/웹사이트 접속 불가
1. 해당 서비스의 상태 페이지 확인
2. 방화벽 규칙 검토
3. DNS 캐시 초기화
